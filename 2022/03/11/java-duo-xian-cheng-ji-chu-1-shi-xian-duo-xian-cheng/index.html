创建线程的两种方法

Thread

public class ThreadStyle extends Thread{

    public static void main(String[] args) {
        new ThreadStyle().start();
    }

    @Override
    public void run() {
        System.out.println("用Thread类实现线程");
    }
}

Runnable

使用Runnable的好处

  • 代码架构角度：增强了程序的扩展性，降低了程序的耦合性（解耦），把线程对象和线程任务进行了分离。

  • 新建线程的损耗：不需要反复 new Thread()，进行线程的创建和销毁。创建的Runnable对象可以重复利用。

  • 避免了单继承的局限性

创建线程

public class RunnableStyle implements Runnable {

    public static void main(String[] args) {
        Thread thread = new Thread(new RunnableStyle());
        thread.start();
    }

    @Override
    public void run() {
        System.out.println("用Runnable方法实现线程");
    }
}

总结：最精准的描述

Oracle官方文档里是说可以分为两类。

准确的讲，创建线程只有一种方式那就是构造Thread类，而实现线程的执行单元有两种方式。

  • 方法一：实现Runnable接口的run方法，并把Runnable实例传给Thread类，调用的就是对象的run方法，执行以下三行代码
  • 方法二：重写Thread的run方法（继承Thread类），就是覆盖了下面run()方法的三行。
@Override
public void run() {
    if (target != null) {
        target.run();
    }
}

典型的错误观点

线程池

错误观点：线程池创建线程也是一种新建线程的方式

其实都是 new Thread() 来创建

static class DefaultThreadFactory implements ThreadFactory {
    private static final AtomicInteger poolNumber = new AtomicInteger(1);
    private final ThreadGroup group;
    private final AtomicInteger threadNumber = new AtomicInteger(1);
    private final String namePrefix;

    DefaultThreadFactory() {
        SecurityManager s = System.getSecurityManager();
        group = (s != null) ? s.getThreadGroup() :
                              Thread.currentThread().getThreadGroup();
        namePrefix = "pool-" +
                      poolNumber.getAndIncrement() +
                     "-thread-";
    }

    public Thread newThread(Runnable r) {
        Thread t = new Thread(group, r,
                              namePrefix + threadNumber.getAndIncrement(),
                              0);
        if (t.isDaemon())
            t.setDaemon(false);
        if (t.getPriority() != Thread.NORM_PRIORITY)
            t.setPriority(Thread.NORM_PRIORITY);
        return t;
    }
}

Callable和FutureTask

  • 错误观点：通过Callable和FutureTask创建线程，也是一种新建线程的方式。
  • 错误观点：无返回值是实现runnable接口，有返回值是实现callable接口，所以callable是新的实现线程的方式。

其实都是实现了Runnable。

public class CallableDemo implements Callable<String> {

    public static void main(String[] args) {
        CallableDemo call = new CallableDemo();
        FutureTask<String> ft = new FutureTask<>(call);
        Thread thread = new Thread(ft);

        thread.start(); // 启动线程

        try {
            System.out.println(ft.get());
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    @Override
    public String call() throws Exception {
        return "callabel方式";
    }
}

定时器

也是实现了Runnable方法。

public class TimmerTaskDemo {

    public static void main(String[] args) {
        Timer timer = new Timer();
        timer.scheduleAtFixedRate(new TimerTask() {
            @Override
            public void run() {
                System.out.println(Thread.currentThread().getName());
            }
        }, 1000, 1000);
    }
}

匿名内部类

只是语法不同。

public class AnonymousInnerClassDemo {

    public static void main(String[] args) {
        // 方法1
        new Thread(){
            @Override
            public void run() {
                System.out.println(Thread.currentThread().getName());
            }
        }.start();

        // 方法2
        new Thread(new Runnable() {
            @Override
            public void run() {
                System.out.println(Thread.currentThread().getName());
            }
        }).start();
    }
}

源码分析

主要看init方法：

  1. 创建此线程，是父线程
  2. ThreadGroup 默认是父线程的ThreadGroup
  3. daemon状态 默认是父线程的daemon状态
  4. 优先级 默认是父线程的优先级
  5. 线程的名称 默认就是Thread-0格式的名称
  6. 线程id是全局递增的，从1开始
class Thread implements Runnable {

    public Thread() {
        init(null, null, "Thread-" + nextThreadNum(), 0);
    }

    private void init(ThreadGroup g, Runnable target, String name,
                      long stackSize) {
        init(g, target, name, stackSize, null, true);
    }

    // 表示不是用java来实现的
    public static native Thread currentThread();

    private void init(ThreadGroup g, Runnable target, String name,
                      long stackSize, AccessControlContext acc,
                      boolean inheritThreadLocals) {
        // 空指针检查
        if (name == null) {
            throw new NullPointerException("name cannot be null");
        }

        // 线程名称赋值
        this.name = name;

        // 创建一个线程的时候，默认父线程就是创建他的那个线程
        // currentThread 就出创建此线程的父线程
        Thread parent = currentThread();
        SecurityManager security = System.getSecurityManager();

        if (g == null) {
            // ...

            // threadGroup没有指定，会自动分配一个线程组，就是父线程的线程组
            // 比如父线程是main线程，此线程的线程组就是 main线程组
            if (g == null) {
                g = parent.getThreadGroup();
            }
        }

        // ...

        this.group = g;
        // 如果没有指定是否为daemon，那么由父线程决定的
        // 父线程是daemon线程，那么此线程也是daemon线程
        this.daemon = parent.isDaemon();
        // 优先级如果没有指定，就跟父线程的优先级保持一致
        this.priority = parent.getPriority();

        // ...

        // 每个线程都有一个线程id，threadId，第一个分配的线程 id是1，以此类推
        tid = nextThreadID();
    }

    private static long threadSeqNumber;
    private static synchronized long nextThreadID() {
        return ++threadSeqNumber;
    }
}
