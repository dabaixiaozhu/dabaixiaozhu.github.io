HashMap

数据结构

JDK 1.8后hashmap的基本的数据结构。

获取key的hash值，再对数组的长度进行取模，根据取模的结果，将key-value放在数组中。

如果某两个key，对应的hash值一样（哈希冲突）：

  • 在JDK 1.8以前，是添加链表，对长链表遍历寻找，性能是O(n)，如果是直接根据array[index]获取到某个元素，性能是O(1)。
  • JDK 1.8以后，如果链表长度超过了8，会将链表转换为红黑树，查找的性能是O(logn)。

关键的内部类

public class HashMap<K,V>  {

    static class Node<K,V> implements Map.Entry<K,V> {
        // key的hash值
        final int hash;
        final K key;
        V value;
        // next的指针，指向单向链表中的下一个节点
        Node<K,V> next;
    }

    // 就是node的一个数组
    transient Node<K,V>[] table;

    // 用set保存所有的k-v对
    transient Set<Map.Entry<K,V>> entrySet;

    // k-v的数量
    transient int size;

    // capacity * loadFactory=threshold，
    // 就是上面的两个成员变量，达到这个数值进行数组扩容
    int threshold;

    // 默认就是负载因子，默认的值是0.75f
    // 也可以进行指定，一般不修改
    final float loadFactor;
}

构造函数

可以直接空的构造函数。

推荐在创建map的时候，传入一个数值。

public class HashMap<K,V>  {

    static final float DEFAULT_LOAD_FACTOR = 0.75f;

    public HashMap() {
        this.loadFactor = DEFAULT_LOAD_FACTOR; // all other fields defaulted
    }

    public HashMap(int initialCapacity) {
        this(initialCapacity, DEFAULT_LOAD_FACTOR);
    }

    // loadFactor 负载因子
    public HashMap(int initialCapacity, float loadFactor) {
        if (initialCapacity < 0)
            throw new IllegalArgumentException("Illegal initial capacity: " +
                                               initialCapacity);
        if (initialCapacity > MAXIMUM_CAPACITY)
            initialCapacity = MAXIMUM_CAPACITY;
        if (loadFactor <= 0 || Float.isNaN(loadFactor))
            throw new IllegalArgumentException("Illegal load factor: " +
                                               loadFactor);
        this.loadFactor = loadFactor;
        // 扩容的数量 capacity * loadFactor
        this.threshold = tableSizeFor(initialCapacity);
    }

    // 返回大于输入参数且最近的2的整数次幂的数。比如10，则返回16
    static final int tableSizeFor(int cap) {
        int n = cap - 1;
        n |= n >>> 1;
        n |= n >>> 2;
        n |= n >>> 4;
        n |= n >>> 8;
        n |= n >>> 16;
        return (n < 0) ? 1 : (n >= MAXIMUM_CAPACITY) ? MAXIMUM_CAPACITY : n + 1;
    }
}

tableSizeFor（重要）

参考 HashMap之tableSizeFor方法图解 - 希夷小道 - 博客园

目的

容量必须是2的整数幂。

给定一个int类型的整数n，如何求出不小于它的最接近的2的整数幂m，比如给定10得出16，给定25得出32？

为什么容量必须是2^n

  1. 容量是2^n时，h & (length - 1) == h % length
  2. 扩容直接旧下标+旧数组长度

普通人的想法

直接对n求以2为底的对数，结果m是double类型，若小数部分为0，则m就是我们要求的指数；

小数部分不为0，则对m向上取整，最后直接求2的m次幂。

jdk没有提供对2求对数的数学公式，只有对自然对数e求对数的公式Math.log(double a)。

只有三步：求对数+取整+求指数。

public static int fun(int n) {
    double m = Math.log(n) / Math.log(2);  
    int m2 = (int) Math.ceil(m);     
    return (int) Math.pow(2, m2);
}

位移来实现

位移的思想

2的整数幂用二进制表示都是最高有效位为1，其余全是0，比如十进制8和32，下图只用了一个字节示意。

举例一个过程图：

初始值：选取任意int类型数字，下图x表示不确定0或者1.

目的是将所有的x变为1。

最后+1，就能进位得到2的整数幂。

右移一位+或运算：有两位变成了1。

右移二位+或运算：有四位变成了1。

右移四位再或运算：有八位变成了1。

右移八位再或运算：有十六位变成了1。

右移十六位再或运算：注意这里不是三十二位全变，而是最高位后面的全变1。

结果+1：

初始容量-1

是为了避免给定容量已经是8,16这样2的幂时，不减1直接移位会导致得到的结果比预期大。

比如预期16得到应该是16，直接移位的话会得到32。

最后一行

最后一行是两个三目运算符，其中之一操作是n+1，很容易理解。

return (n < 0) ? 1 : (n >= MAXIMUM_CAPACITY) ? MAXIMUM_CAPACITY : n + 1;

总结

这个方法之所以高效，是因为移位运算和或运算都属于比较底层的操作，代码的数量不会比最终的指令数多，也就是通过几个简单操作实现了目的。

hash相关优化

hash值算法的优化：高低16位异或

异或就是相同为0，不同为1.

hash方法计算出了key的hash值。

public class HashMap<K,V> {

    public V put(K key, V value) {
        return putVal(hash(key), key, value, false, true);
    }


    static final int hash(Object key) {
        int h;
        return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
    }
}
h ^ (h >>> 16)

因为哈希寻址时，也有一个位运算(n - 1) & hash ，如果数组很小，一般都是用低16位在进行运算，那么哈希值的高 16 位没参与到运算中去。

所以需要将高16位和低16位进行一个异或运算（相同为0，不同为1），将高16位和低16位的特征 都参与到了定位index的运算，从而降低hash冲突。

1111 1111 1111 1111 1111 1010 0111 1100
0000 0000 0000 0000 1111 1111 1111 1111
1111 1111 1111 1111 0000 0101 1000 0011

hash寻址算法的优化：位运算代替取模（与2的n次方有关）

提升性能的优化点：直接取模的性能相对较低，采用位运算的方法。

数组初始值，以及每次扩容后的大小，都是2的n次方，就可以保证(n - 1) & hash（就是数组对应的index），跟hash % 数组.length是一样的效果。

public class HashMap<K,V> {

    public V put(K key, V value) {
        return putVal(hash(key), key, value, false, true);
    }


    final V putVal(int hash, K key, V value, boolean onlyIfAbsent,
                   boolean evict) {
        Node<K,V>[] tab; Node<K,V> p; int n, i;
        // 初始化数组
        if ((tab = table) == null || (n = tab.length) == 0)
            // 初始化数组的长度，默认是16，负载因子是0.75，threshold是12
            n = (tab = resize()).length;
        // 哈希寻址算法
        // 这里用 与操作（必须都是1，才是1，否则就是0）来达到取模的效果
        // 位运算，性能很高
        if ((p = tab[i = (n - 1) & hash]) == null)
            tab[i] = newNode(hash, key, value, null);
        // ...
        return null;
    }
}

hash冲突时的处理（put方法）

两个key的hash值一样的概率很低。

更常见的哈希冲突是，两个key的hash值不一样，但是通过寻址算法，定位到了数组的同一个index上，默认情况用单向链表来处理。

注意转换的条件有两个，见下面的标黄内容：

  1. 链表长度>8
  2. 数组长度大于64

使用链表

public class HashMap<K,V> {

    static final int TREEIFY_THRESHOLD = 8;

    final V putVal(int hash, K key, V value, boolean onlyIfAbsent,
                   boolean evict) {
        Node<K,V>[] tab; Node<K,V> p; int n, i;
        if ((tab = table) == null || (n = tab.length) == 0)
            n = (tab = resize()).length;

        // 如果哈希定位到的位置i是空的，就是之前这个位置未放入元素
        if ((p = tab[i = (n - 1) & hash]) == null)
            // 放一个node在此位置即可
            tab[i] = newNode(hash, key, value, null);

        // 之前此位置有元素
        else {
            Node<K,V> e; K k;
            // p就是指向i位置的node
            // i位置key相同，需要进行覆盖旧的value
            if (p.hash == hash &&
                ((k = p.key) == key || (key != null && key.equals(k))))
                // e用于下面判断
                e = p;

            // 表示不是同一个key（也就不是同一个哈希值）
            // 这个位置如果是 红黑树 的情况
            else if (p instanceof TreeNode)
                e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);

            // 表示不是同一个key（也就不是同一个哈希值）
            // 这里是链表的处理
            else {
                // binCount 就是当前链表长度
                for (int binCount = 0; ; ++binCount) {
                    if ((e = p.next) == null) {
                        // 拼接链表 核心代码
                        p.next = newNode(hash, key, value, null);

                        // 转为红黑树的两个条件
                        // 1.binCount >=7 即达到9个节点，9个节点的原因是，这里传进来的是p.next，当binCount=0时，put第二个元素
                        // 2.并且数组长度大于64，详见下面的treeifyBin方法
                        if (binCount >= TREEIFY_THRESHOLD - 1) // -1 for 1st
                            treeifyBin(tab, hash);
                        break;
                    }

                    if (e.hash == hash &&
                        ((k = e.key) == key || (key != null && key.equals(k))))
                        break;
                    p = e;
                }
            }

            // key的覆盖新的值
            if (e != null) { // existing mapping for key
                V oldValue = e.value;
                if (!onlyIfAbsent || oldValue == null)
                    e.value = value;
                afterNodeAccess(e);
                return oldValue;
            }
        }
        ++modCount;
        if (++size > threshold)
            resize();
        afterNodeInsertion(evict);
        return null;
    }
}

JDK 1.8引入红黑树优化hash冲突

红黑树和链表转换的阈值

链表转红黑树是链表长度达到阈值是8（即超过8，要到达9）：发生hash碰撞8次的几率为百万分之6，8够用了。

红黑树转链表阈值为6：如果hash碰撞次数在8附近徘徊，会一直发生链表和红黑树的互相转化，为了预防这种情况的发生，设置为6。

链表转红黑树

链表长度太长的话，get()操作的时间复杂度可能是O(n)，正常数组索引直接定位的方式 O(1)。

用红黑树的话，get()操作 O(logn)。

public class HashMap<K,V> {
    static final int MIN_TREEIFY_CAPACITY = 64;

    final void treeifyBin(Node<K,V>[] tab, int hash) {
        int n, index; Node<K,V> e;
        // 判断数组长度是否小于64,小于则进行扩容,否则转红黑树
        if (tab == null || (n = tab.length) < MIN_TREEIFY_CAPACITY)
            resize();
        // 通过哈希值进行寻址
        else if ((e = tab[index = (n - 1) & hash]) != null) {
            TreeNode<K,V> hd = null, tl = null;
            // do while循环执行完了以后：
            // 先是将单向链表转换为了TreeNode类型组成的一个双向链表
            do {
                // 传入链表头结点 转成了红黑树
                // p就是红黑树的根节点
                TreeNode<K,V> p = replacementTreeNode(e, null);
                if (tl == null)
                    hd = p;
                else {
                    p.prev = tl;
                    tl.next = p;
                }
                tl = p;
            } while ((e = e.next) != null);

            if ((tab[index] = hd) != null)
                // 这里转为红黑树
                hd.treeify(tab);
        }
    }

    // 红黑树
    static final class TreeNode<K,V> extends LinkedHashMap.Entry<K,V> {
        TreeNode<K,V> parent;  // red-black tree links
        TreeNode<K,V> left;
        TreeNode<K,V> right;
        TreeNode<K,V> prev;    // needed to unlink next upon deletion
        // 是红色还是黑色
        boolean red;
        TreeNode(int hash, K key, V val, Node<K,V> next) {
            super(hash, key, val, next);
        }

        // 很多操作树的方法
        // ...
    }
}

扩容机制

HashMap底层是使用数组

JDK1.7扩容时的问题

JDK 1.7：扩容2倍，rehash过程是基于key的hash值重新寻找位置，这样会导致所有数值重新排列。

原先同一位置的key，可能不同位置

  • 数组长度=16

  • 数组长度扩容之后 = 32

JDK 1.8的优化：高性能rehash算法

rehash算法

JDK 1.8以后，扩容一定是2 ^ n，用的是 & （两个位都为1时，结果才为1）来实现hash寻址的算法。

每次扩容之后，hash值要么是在原本的位置，要么是变成了 原来的位置 + 旧数组的长度。

比如：index(5) + oldCap(16) = 21

这样可以将一个位置的多个key，分散到新数组的不同位置，减少哈希冲突。

原本的两个数：存在哈希碰撞的问题。

n - 1    0000 0000 0000 0000 0000 0000 0000 1111 （数组长度-1）
hash1    1111 1111 1111 1111 0000 1111 0000 0101
&结果     0000 0000 0000 0000 0000 0000 0000 0101  = 5（index = 5的位置）
n - 1    0000 0000 0000 0000 0000 0000 0000 1111
hash2    1111 1111 1111 1111 0000 1111 0001 0101
&结果     0000 0000 0000 0000 0000 0000 0000 0101 = 5（index = 5的位置）

如果数组的长度扩容之后 = 32

n-1      0000 0000 0000 0000 0000 0000 0001 1111
hash1    1111 1111 1111 1111 0000 1111 0000 0101
&结果     0000 0000 0000 0000 0000 0000 0000 0101 = 5（index = 5的位置）
n-1      0000 0000 0000 0000 0000 0000 0001 1111
hash2    1111 1111 1111 1111 0000 1111 0001 0101
&结果     0000 0000 0000 0000 0000 0000 0001 0101 = 21（index = 21的位置）

    final Node<K,V>[] resize() {
        // ...

        if (oldCap > 0) {
            if (oldCap >= MAXIMUM_CAPACITY) {
                threshold = Integer.MAX_VALUE;
                return oldTab;
            }
            // 新数组的大小是老数组的2倍
            else if ((newCap = oldCap << 1) < MAXIMUM_CAPACITY &&
                     oldCap >= DEFAULT_INITIAL_CAPACITY)
                newThr = oldThr << 1; // double threshold
        }

        // ...

        // 将原来数组中的所有元素都 copy进新的数组
        if(oldTab != null){
            for (int j = 0; j < oldCap; j++) {
                Entry e;

                if((e = oldTab[j]) != null){
                    oldTab[j] = null;

                    // 说明还没有成链表，数组上只有一个值
                    if(e.next == null){
                        // 重新计算 数组索引值，直接插入数组
                        newTable[e.h & (newCap-1)] = e;
                    }

                    // 这个位置是红黑树
                    else if (e instanceof TreeNode)
                        // 遍历这颗红黑树，将里面每个节点都进行hash寻址，找到新数组的某个位置
                        ((TreeNode<K,V>)e).split(this, newTab, j, oldCap);


                    // 如果不是树，只是链表,即长度还没有大于 8 进化成树
                    // 重新组织一下链表
                    else{
                        // ...

                        // 新的index 等于原来的 index+oldCap
                        else {
                            hiTail.next = null;
                            newTable[j+oldCap] = hiHead;
                        }
                    }
                }
            }
        }

常见操作

get与操作

public class HashMap<K,V> {

    public V get(Object key) {
        Node<K,V> e;
        return (e = getNode(hash(key), key)) == null ? null : e.value;
    }

    final Node<K,V> getNode(int hash, Object key) {
        Node<K,V>[] tab; Node<K,V> first, e; int n; K k;
        if ((tab = table) != null && (n = tab.length) > 0 &&
            (first = tab[(n - 1) & hash]) != null) {

            // 从数组中直接取出元素
            if (first.hash == hash && // always check first node
                ((k = first.key) == key || (key != null && key.equals(k))))
                return first;
            if ((e = first.next) != null) {
                // 红黑树的情况
                if (first instanceof TreeNode)
                    return ((TreeNode<K,V>)first).getTreeNode(hash, key);
                // 链表的情况
                do {
                    if (e.hash == hash &&
                        ((k = e.key) == key || (key != null && key.equals(k))))
                        return e;
                } while ((e = e.next) != null);
            }
        }
        return null;
    }
}

remove操作

public class HashMap<K,V> {

    public V remove(Object key) {
        Node<K,V> e;
        return (e = removeNode(hash(key), key, null, false, true)) == null ?
            null : e.value;
    }

    final Node<K,V> removeNode(int hash, Object key, Object value,
                               boolean matchValue, boolean movable) {
        Node<K,V>[] tab; Node<K,V> p; int n, index;
        if ((tab = table) != null && (n = tab.length) > 0 &&
            (p = tab[index = (n - 1) & hash]) != null) {
            Node<K,V> node = null, e; K k; V v;

            // 这里是找出元素
            // 数组直接取出来
            if (p.hash == hash &&
                ((k = p.key) == key || (key != null && key.equals(k))))
                node = p;
            else if ((e = p.next) != null) {
                // 红黑树拿出node
                if (p instanceof TreeNode)
                    node = ((TreeNode<K,V>)p).getTreeNode(hash, key);
                // 链表直接遍历
                else {
                    do {
                        if (e.hash == hash &&
                            ((k = e.key) == key ||
                             (key != null && key.equals(k)))) {
                            node = e;
                            break;
                        }
                        p = e;
                    } while ((e = e.next) != null);
                }
            }

            // 这里是删除元素
            if (node != null && (!matchValue || (v = node.value) == value ||
                                 (value != null && value.equals(v)))) {
                // 红黑树移除node
                if (node instanceof TreeNode)
                    ((TreeNode<K,V>)node).removeTreeNode(this, tab, movable);
                // 链表表头
                else if (node == p)
                    tab[index] = node.next;
                // 链表中间的位置
                else
                    p.next = node.next;
                ++modCount;
                --size;
                afterNodeRemoval(node);
                return node;
            }
        }
        return null;
    }
}

JDK1.7下 HashMap在高并发下会出现死循环

环形链表

参考 JAVA HASHMAP的死循环

具体见流程图，总共三次循环。

// 从旧数组转移到新数组，注意是反过来移动
void transfer(Entry[] newTable) {
    // 获取到旧数组
    Entry[] src = table;
    // 新table的容量
    int newCapacity = newTable.length;
    // 遍历原数组
    for (int j = 0; j < src.length; j++) {
        // 数组的每个节点
        Entry<K,V> e = src[j];
        if (e != null) {
            // 清空旧数组的元素
            src[j] = null;
            // 遍历完链表
            do {
                Entry<K,V> next = e.next; 

                // i表示在新数组（默认扩容2倍）里的新位置
                // 线程1执行到此位置，换为线程2，线程2执行完所有流程后，再切换为线程1
                int i = indexFor(e.hash, newCapacity);
                // 进行转移，头插法
                e.next = newTable[i];
                newTable[i] = e;
                e = next;
            } while (e != null);
        }
    }
}

头插法图解

在transfer的过程中，比如轮到了图中的 <k1,v1>这串链表的位置，而新位置之前已经插入了一个 <k4,v4>链表。

死循环

get(k5)，因为环形链表里没有k5的值，所以会导致死循环。

<k3,v3>没有纳入链表中

1.8后的解决

JDK 1.8以后的hashmap，采用了红黑树的数据结构，优化了数组扩容的过程，就不会再有死循环的问题。

LinkedHashMap

插入

LinkedHashMap的put，其实还是会跳转到HashMap中的put，顺着往下看。

插入时基于双向链表维护顺序。

一开始一个节点，所以tail和head两个指针，都会指向p。

public class LinkedHashMapDemo {
    public static void main(String[] args) {
        LinkedHashMap<Integer, String> map = new LinkedHashMap<>();
        map.put(1,"张三");
    }
}

public class LinkedHashMap<K,V> {

    // 重写了 HashMap中的 newNode 方法
    // hash：key的哈希值
    // e：next，表示指向的下一个节点
    Node<K,V> newNode(int hash, K key, V value, Node<K,V> e) {
        // 将节点封装为一个LinkedHashMap.Entry对象
        LinkedHashMap.Entry<K,V> p =
            new LinkedHashMap.Entry<K,V>(hash, key, value, e);
        // 使用linkNodeLast(p) 将节点挂到一个链表里
        linkNodeLast(p);
        return p;
    }

    private void linkNodeLast(LinkedHashMap.Entry<K,V> p) {
        // last指针
        LinkedHashMap.Entry<K,V> last = tail;
        tail = p;
        // 相当于没有节点
        if (last == null)
            head = p;
        else {
            // 双向链表，往原本的last后加一个p
            p.before = last;
            last.after = p;
        }
    }
}

覆盖一个值

覆盖一个值，不会改变他的顺序。

accessOrder，默认是false，表示覆盖key值，不会改变在链表里的顺序。

如果accessOrder是true，get、put操作会让key挪动到链表的尾部。

public class HashMap<K,V> {

    final V putVal(int hash, K key, V value, boolean onlyIfAbsent,
                   boolean evict) {
        // ...

        if (e != null) { // existing mapping for key
            V oldValue = e.value;
            if (!onlyIfAbsent || oldValue == null)
                e.value = value;
            afterNodeAccess(e);
            return oldValue;
        } 
    }
}

public class LinkedHashMap<K,V> {

    final boolean accessOrder;

    void afterNodeAccess(Node<K,V> e) { // move node to last
        LinkedHashMap.Entry<K,V> last;
        if (accessOrder && (last = tail) != e) {
            // ...
        }
    }

    public LinkedHashMap(int initialCapacity,
                         float loadFactor,
                         boolean accessOrder) {
        super(initialCapacity, loadFactor);
        this.accessOrder = accessOrder;
    }
}

TreeMap

TreeMap，底层基于红黑树，可以按照key的自然顺序来排序，既然人家是按照key的大小来进行排序和迭代输出的

人家肯定是用了一个按照key大小排序的数据结构，我们可以想到的就是红黑树，所以他底层是基于红黑树来实现的按key排序就可以了。
