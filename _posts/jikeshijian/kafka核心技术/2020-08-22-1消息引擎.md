# <a name="1">[概述](#begin)</a>

- Messaging System，消息引擎
- Apache Kafka 是一款开源的消息引擎系统

# <a name="2">[作用](#begin)</a>

- 消息引擎传输的对象是消息；
- 如何传输消息属于消息引擎设计机制的一部分

# <a name="3">[传输协议](#begin)</a>

- 点对点模型
  - 系统 A 发送的消息只能被系统 B 接收，其他任何系统都不能读取 A 发送的消息
- 发布 / 订阅模型
  - 有一个主题（Topic）的概念
  - 发送方也称为发布者（Publisher），接收方称为订阅者（Subscriber）
  - 和点对点模型不同：这个模型可能存在多个发布者向相同的主题发送消息，而订阅者也可能存在多个，它们都能接收到相同主题的消息

# <a name="4">[为何使用消息引擎](#begin)</a>

削峰填谷

- 指缓冲上下游瞬时突发流量，使其更平滑
- 对于发送能力很强的上游系统，如果没有消息引擎的保护，“脆弱”的下游系统可能会直接被压垮导致全链路服务“雪崩”

引入kafka后

- 流程
  - 上游订单服务不再直接与下游子服务进行交互
  - 当新订单生成后它仅仅是向 Kafka Broker 发送一条订单消息即可
  - 下游的各个子服务订阅 Kafka 中的对应主题，并实时从该主题的各自分区（Partition）中获取到订单消息进行处理，从而实现了上游订单服务与下游订单处理服务的解耦
- 当出现秒杀业务
  - Kafka 能够将瞬时增加的订单流量全部以消息形式保存在对应的主题中
  - 既不影响上游服务的 TPS，同时也给下游子服务留出了充足的时间去消费它们
